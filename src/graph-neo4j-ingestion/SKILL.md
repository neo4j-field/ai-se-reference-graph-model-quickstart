# Graph Neo4j Ingestion

Loads generated graph data into a Neo4j database. Two modes are available:

- **Mode A: MCP Tools** — Claude directly executes Cypher via the connected Neo4j MCP tools.
  Best when the user's Neo4j instance is already connected to Claude.
- **Mode B: Python Script** — A standalone `ingest_data.py` script the user can run
  themselves against any Neo4j instance. Best when the user wants to run it outside Claude,
  in CI/CD, or against a remote database.

## When to Use

- User has generated CSV data (from graph-data-generator) and wants it in Neo4j
- User has a schema and wants to create constraints and indexes
- User asks to "load", "ingest", "import", or "push" data into Neo4j
- User asks for an ingestion script they can run themselves

## Choosing the Mode

Ask the user (or infer from context):

- If **Neo4j MCP tools are connected** and the user says "load this into Neo4j" or
  "ingest this data" → default to **Mode A** (direct MCP ingestion).
- If the user says "give me a script", "I want to run it myself", or doesn't have MCP
  connected → use **Mode B** (Python script).
- If both are possible, briefly offer the choice.

## Mode A: Direct Ingestion via MCP Tools

### Prerequisites
- `write_neo4j_cypher` and `read_neo4j_cypher` MCP tools must be connected.

### Step 1: Identify the Data Source

Read the manifest from the data generator:
```bash
cat /home/claude/graph_data/manifest.json
```

### Step 2: Confirm with the User

Before writing to their database, confirm:
- Whether they want to clear existing data first
- Whether they want constraints/indexes created

### Step 3: Create Constraints

For each node label, use `write_neo4j_cypher`:

```cypher
CREATE CONSTRAINT <label>_id IF NOT EXISTS
FOR (n:<Label>) REQUIRE n._id IS UNIQUE
```

Run one constraint per tool call.

### Step 4: Ingest Node Data

For each node CSV:
1. Read the CSV with Python (`csv.DictReader`)
2. Batch into chunks of 500 rows
3. For each batch, call `write_neo4j_cypher`:

```cypher
UNWIND $rows AS row
CREATE (n:<Label>)
SET n = row
```

Pass rows as params: `{"rows": batch}`

### Step 5: Ingest Relationship Data

For each relationship CSV:
1. Read the CSV, batch into 500-row chunks
2. For each batch:

```cypher
UNWIND $rows AS row
MATCH (a:<FromLabel> {_id: row._from_id})
MATCH (b:<ToLabel> {_id: row._to_id})
CREATE (a)-[r:<REL_TYPE>]->(b)
SET r.prop1 = row.prop1, r.prop2 = row.prop2
```

Build the SET clause dynamically from the manifest's property list.
Omit `_from_id` and `_to_id` from the SET.

### Step 6: Verify

```cypher
MATCH (n) RETURN labels(n)[0] AS label, count(n) AS count ORDER BY count DESC
```

```cypher
MATCH ()-[r]->() RETURN type(r) AS type, count(r) AS count ORDER BY count DESC
```

### Step 7: Report Results

Summarize node counts, relationship counts, and any errors.

## Mode B: Standalone Python Ingestion Script

### Overview

The skill bundles `assets/ingest_data.py` — a self-contained Python script that:
- Reads `manifest.json` and all CSV files from a data directory
- Connects to Neo4j using the official `neo4j` Python driver
- Creates uniqueness constraints
- Batch-loads nodes and relationships using UNWIND
- Verifies the result with count queries
- Supports `--drop-existing` to clear data first

### Step 1: Set Up the Virtual Environment

Reuse the same venv created by the data generator, or create one:

```bash
if [ ! -d "/home/claude/graph_venv" ]; then
    python3 -m venv /home/claude/graph_venv
fi
/home/claude/graph_venv/bin/pip install neo4j -q
```

### Step 2: Copy and Present the Script

```bash
cp /path/to/skill/assets/ingest_data.py /home/claude/ingest_data.py
cp /home/claude/ingest_data.py /mnt/user-data/outputs/ingest_data.py
```

Present the file to the user so they can download it.

### Step 3: Show the User How to Run It

Tell the user:

```bash
# Activate the virtual environment (or use the venv python directly)
source /path/to/graph_venv/bin/activate

# Run the ingestion
python ingest_data.py ./graph_data \
    --uri bolt://localhost:7687 \
    --user neo4j \
    --password <your-password> \
    --database neo4j \
    --batch-size 500
```

Or with the venv python directly (no activate needed):
```bash
/path/to/graph_venv/bin/python3 ingest_data.py ./graph_data \
    --uri bolt://localhost:7687 \
    --user neo4j \
    --password <your-password>
```

#### Script Arguments

| Argument | Default | Description |
|---|---|---|
| `data_dir` | (required) | Directory with manifest.json and CSVs |
| `--uri` | bolt://localhost:7687 | Neo4j connection URI |
| `--user` | neo4j | Neo4j username |
| `--password` | (required) | Neo4j password |
| `--database` | neo4j | Target database name |
| `--batch-size` | 500 | Rows per UNWIND batch |
| `--drop-existing` | false | Clear all data before loading |

### Step 4: Offer to Run It in Claude

If the user provides their Neo4j credentials and wants Claude to run the script,
execute it via bash:

```bash
/home/claude/graph_venv/bin/python3 /home/claude/ingest_data.py \
    /home/claude/graph_data \
    --uri <URI> \
    --user <USER> \
    --password <PASSWORD> \
    --database <DB>
```

**Important**: Handle credentials carefully. Don't log or echo the password.

## Handling Large Datasets

For datasets > 5,000 rows per type:
- Use batch sizes of 500 rows
- Show progress ("Loading batch 3/20 for Person nodes...")
- Continue on batch failure, report at end

For datasets > 50,000 rows:
- Warn the user it may take a while
- Suggest `neo4j-admin import` or Data Importer for very large datasets
- Use smaller batches (250) if timeouts occur

## Clearing Existing Data

Both modes support clearing data:

- **Mode A**: Run `MATCH (n) WITH n LIMIT 10000 DETACH DELETE n` in a loop
- **Mode B**: Use the `--drop-existing` flag

**Always confirm with the user before clearing.** Suggest it only for dev/test databases.

## Error Handling

Common issues and fixes:
- **"already exists"** on constraints → harmless, IF NOT EXISTS handles it
- **Connection refused** → check URI, ensure Neo4j is running
- **Authentication failed** → check username/password
- **Timeout on large batches** → reduce `--batch-size` to 250 or 100
- **MATCH finds no nodes for relationships** → ensure nodes were loaded first

## Important Notes

- Mode A uses Neo4j MCP tools only — no Python driver needed
- Mode B uses the `neo4j` Python driver — no MCP connection needed
- Both modes use the same manifest.json and CSV format from graph-data-generator
- The `_id` column is a synthetic UUID used as the merge key. Users can remove it later
- The same virtual environment (`/home/claude/graph_venv`) is shared between the
  data generator (faker) and this script (neo4j driver)
